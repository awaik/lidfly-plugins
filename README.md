# Плагин LidFly для Codex

LidFly подключает Codex к рекламе, аналитике, сайтам и рабочим проектам через один защищённый MCP-сервер. После установки можно просить Codex проверить статистику, найти проблемы в кампаниях, подготовить изменения или помочь с контентом — без ручного копирования API-ключей в конфигурацию.

Сейчас плагин устанавливается через Codex CLI. Установщик без терминала для macOS и Windows ещё готовится.

## Что можно делать

После подключения Codex сможет работать с теми сервисами и проектами, к которым у вашего аккаунта LidFly уже есть доступ:

- анализировать кампании и статистику Яндекс Директа, VK Ads и Авито Рекламы;
- работать с отчётами Яндекс Метрики и другими подключёнными источниками;
- помогать с объявлениями, ставками, аудитами и рекламными материалами;
- создавать и редактировать сайты и контент в LidFly;
- использовать проекты, задачи и контекст из «Пространств»;
- проверять доступные подключения и статус подписки.

Набор возможностей зависит от подключённых кабинетов, прав пользователя и тарифа LidFly. Если Codex предлагает изменить данные или настройки рекламы, внимательно проверьте действие перед подтверждением.

## Как это работает

1. Вы устанавливаете плагин LidFly в Codex.
2. Плагин добавляет подключение к `https://lidfly.ru/mcp/v3`.
3. Codex открывает вход LidFly в браузере.
4. Вы входите по email и подтверждаете подключение.
5. В новом чате Codex получает только те инструменты и проекты, которые доступны вашему аккаунту.

В самом репозитории нет пользовательских API-ключей, паролей или токенов рекламных кабинетов. Плагин хранит только публичное описание и адрес MCP-сервера; авторизация выполняется через OAuth LidFly.

## Установка через Codex CLI

Это основной доступный способ установки.

```sh
codex plugin marketplace add awaik/lidfly-plugins
codex plugin add lidfly@lidfly
```

После выполнения команд перезапустите Codex, пройдите вход LidFly и начните новый чат.

Проверить подключение можно так:

```sh
codex plugin marketplace list
codex plugin list --json
codex mcp list
```

В списке должен появиться плагин `lidfly`, а для MCP-сервера — адрес `https://lidfly.ru/mcp/v3`.

## Установка без терминала — скоро

Установщик для macOS и Windows ещё не опубликован. Пока используйте установку через Codex CLI выше.

Когда установщик станет доступен, он будет добавлять только marketplace LidFly. Установку самого плагина и вход в аккаунт вы по-прежнему будете отдельно подтверждать в штатном интерфейсе Codex.

## С чего начать

Попробуйте написать в новом чате:

- «Проверь статус моей подписки LidFly»;
- «Покажи доступные аккаунты и проекты»;
- «Покажи мои кампании в Яндекс Директе»;
- «Проанализируй статистику за последние 30 дней и найди точки роста»;
- «Проверь сайт и предложи улучшения».

Codex сам запросит нужные инструменты у LidFly. Подключать отдельный MCP-сервер для каждого рекламного кабинета не требуется.

## Если плагин не появился

1. Убедитесь, что marketplace виден в `codex plugin marketplace list`.
2. Проверьте установку через `codex plugin list --json`.
3. Полностью закройте Codex, откройте его снова и создайте новый чат.
4. Если LidFly раньше подключался вручную, старый custom MCP может дублировать сервер из плагина. Отключайте его только после успешной проверки нового подключения.
5. В корпоративном Workspace установка персональных marketplaces может быть запрещена администратором. Плагин не обходит эти ограничения.

Если marketplace нужно обновить, выполните:

```sh
codex plugin marketplace upgrade lidfly
```

## Удаление

Удалить плагин можно командой:

```sh
codex plugin remove lidfly@lidfly
```

Чтобы также убрать marketplace LidFly из локальной конфигурации:

```sh
codex plugin marketplace remove lidfly
```

Доступ выданного OAuth-подключения при необходимости можно отдельно отозвать в LidFly: **Настройки → Подключённые ИИ-клиенты**.

## Что находится в репозитории

```text
.
├── .agents/plugins/marketplace.json          каталог marketplace LidFly
├── plugins/lidfly/
│   ├── .codex-plugin/plugin.json             описание плагина
│   ├── .mcp.json                             подключение к LidFly MCP
│   └── assets/                               иконки и логотипы
└── LICENSE                                   лицензия MIT
```

Плагин не запускает отдельный локальный сервер и не устанавливает зависимости: он подключает Codex к удалённому MCP LidFly по HTTPS.
